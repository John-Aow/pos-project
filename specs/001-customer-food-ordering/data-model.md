# Data Model: Customer Food Ordering

## Menu Item

Existing catalog entity used by customer ordering.

Fields:
- `id`: stable menu item identifier
- `name`: customer-visible name
- `description`: customer-visible description
- `price_cents`: integer price in cents
- `stock_quantity`: current stock quantity
- `is_active`: whether the catalog item may be sold
- `is_available`: derived orderability state, true only when active and in stock
- `category_id`: category reference

Rules:
- Inactive items are not orderable.
- Items with `stock_quantity <= 0` are not orderable.
- The backend rechecks availability during order submission.

## Cart

Frontend-only in-progress customer selection before submission.

Fields:
- `items`: one or more cart items
- `order_total_cents`: sum of all line totals
- `idempotency_key`: generated before submission and reused while retry status is uncertain

Rules:
- Empty carts cannot be submitted.
- Totals are preview totals only until backend submission validates prices and availability.

## Cart Item

Frontend-only selected menu item.

Fields:
- `menu_item_id`: selected menu item ID
- `name`: copied menu item name for display
- `price_cents`: current displayed price
- `quantity`: positive integer
- `note`: optional customer note
- `line_total_cents`: `price_cents * quantity`
- `is_orderable`: current frontend orderability flag

Rules:
- Quantity must be at least 1.
- Notes must be bounded in length before submission.
- Unavailable items cannot be added.

## Order

Persisted customer order record.

Fields:
- `id`: internal order identifier
- `order_number`: customer-visible unique order number
- `status`: pending, preparing, served, or canceled
- `submitted_at`: server timestamp
- `idempotency_key`: unique key for duplicate submission prevention
- `total_cents`: persisted total used for confirmation and billing
- `lines`: submitted order lines

Rules:
- Order number is generated by the backend.
- Order total is calculated by the backend from submission-time menu prices.
- Submitted orders are append-only for customer ordering; they must not delete catalog or history.

## Order Line

Persisted line item on an order.

Fields:
- `id`: internal line identifier
- `order_id`: parent order ID
- `menu_item_id`: ordered menu item ID
- `name`: menu item name snapshot at submission
- `price_cents`: price snapshot at submission
- `quantity`: positive integer
- `note`: optional item-specific note
- `line_total_cents`: persisted line total

Rules:
- Line total equals price snapshot multiplied by quantity.
- Notes are stored with the line for staff preparation.

## Bill

Financial record associated with the submitted order.

Fields:
- `id`: internal bill identifier
- `order_id`: parent order ID
- `total_cents`: total due for the order
- `status`: open, paid, canceled, or split
- `created_at`: server timestamp

Rules:
- Payment collection is out of scope for this feature.
- Bill total must match the order total at creation.

## Incoming Order List

Staff-facing projection of active orders.

Fields:
- `order_number`
- `status`
- `submitted_at`
- `total_cents`
- `lines`

Rules:
- Newly submitted orders appear with `pending` status.
- The list includes item quantities and notes.
